Authentication flows

A maioria dos aplicativos exige que um usuário se autentique de alguma forma para ter acesso aos dados associados a um usuário ou outro conteúdo privado. Normalmente, o fluxo ficará assim:

  • O usuário abre o aplicativo.
  • O aplicativo carrega algum estado de autenticação do armazenamento persistente (por exemplo, AsyncStorage).
  • Quando o estado foi carregado, o usuário é apresentado com telas de autenticação ou o aplicativo principal, dependendo se o estado de autenticação válido foi carregado.
  • Quando o usuário sai, limpamos o estado de autenticação e os enviamos de volta para as telas de autenticação.

Nota: nós dizemos "telas de autenticação" porque geralmente há mais de um. Você pode ter uma tela principal com um campo de nome de usuário e senha, outro para "Esqueci a senha" e outro conjunto para sign up.

Configurar os nossos navegadores

import { createAppContainer, createSwitchNavigator } from 'react-navigation';
import { createStackNavigator } from 'react-navigation-stack';

// Implementation of HomeScreen, OtherScreen, SignInScreen, AuthLoadingScreen
// goes here.

const AppStack = createStackNavigator({ Home: HomeScreen, Other: OtherScreen });
const AuthStack = createStackNavigator({ SignIn: SignInScreen });

export default createAppContainer(
  createSwitchNavigator(
    {
      AuthLoading: AuthLoadingScreen,
      App: AppStack,
      Auth: AuthStack,
    },
    {
      initialRouteName: 'AuthLoading',
    }
  )
);

Você pode não estar familiarizado com SwitchNavigator ainda. O objetivo de SwitchNavigator é apenas mostrar uma tela por vez. Por padrão, ele não controla as ações de retorno e redefine as rotas para seu estado padrão quando você é desligado. Esse é o comportamento exato que desejamos do fluxo de autenticação: quando os usuários fazem login, queremos descartar o estado do fluxo de autenticação e desmontar todas as telas, e quando pressionamos o botão Voltar do hardware, não esperamos poder para voltar ao fluxo de autenticação. Podemos alternar entre rotas no SwitchNavigator utilizando navigate. Você pode ler que mais sobre o SwitchNavigator no API referência.

Nós definimos o initialRouteName para 'AuthLoading' porque buscaremos nosso estado de autenticação do armazenamento persistente dentro daquele componente da tela.

Implementar nossa tela de carregamento de autenticação

import React from 'react';
import {
  ActivityIndicator,
  AsyncStorage,
  StatusBar,
  StyleSheet,
  View,
} from 'react-native';

class AuthLoadingScreen extends React.Component {
  componentDidMount() {
    this._bootstrapAsync();
  }

  // Fetch the token from storage then navigate to our appropriate place
  _bootstrapAsync = async () => {
    const userToken = await AsyncStorage.getItem('userToken');

    // This will switch to the App screen or Auth screen and this loading
    // screen will be unmounted and thrown away.
    this.props.navigation.navigate(userToken ? 'App' : 'Auth');
  };

  // Render any loading content that you like here
  render() {
    return (
      <View>
        <ActivityIndicator />
        <StatusBar barStyle="default" />
      </View>
    );
  }
}

Preencha outros componentes

Nossas rotas App e Auth são ambas navegadores de pilha, mas você pode fazer o que quiser aqui. Como mencionado acima, você provavelmente quer que sua rota de autenticação seja uma pilha para redefinição de senha, inscrição, etc. Da mesma forma para o seu aplicativo, você provavelmente tem mais de uma tela. Nós não vamos falar sobre como implementar as entradas de texto e botões para a tela de autenticação, que está fora do escopo de navegação. Vamos apenas preencher algum conteúdo.

Auth Flow

class SignInScreen extends React.Component {
  static navigationOptions = {
    title: 'Please sign in',
  };

  render() {
    return (
      <View>
        <Button title="Sign in!" onPress={this._signInAsync} />
      </View>
    );
  }

  _signInAsync = async () => {
    await AsyncStorage.setItem('userToken', 'abc');
    this.props.navigation.navigate('App');
  };
}

class HomeScreen extends React.Component {
  static navigationOptions = {
    title: 'Welcome to the app!',
  };

  render() {
    return (
      <View>
        <Button title="Show me more of the app" onPress={this._showMoreApp} />
        <Button title="Actually, sign me out :)" onPress={this._signOutAsync} />
      </View>
    );
  }

  _showMoreApp = () => {
    this.props.navigation.navigate('Other');
  };

  _signOutAsync = async () => {
    await AsyncStorage.clear();
    this.props.navigation.navigate('Auth');
  };
}

// More code like OtherScreen omitted for brevity

Isso é tudo o que existe para isso. If you're interested in animating the switch between screens, you can read about createAnimatedSwitchNavigator in the API reference.
