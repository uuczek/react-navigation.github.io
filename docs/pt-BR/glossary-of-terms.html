Glossary of terms

Esta é uma nova seção da documentação e falta muitos termos! Please submit a pull request or an issue with a term that you think should be explained here.

Header

Também conhecido como cabeçalho de navegação, barra de navegação, navbar e provavelmente muitas outras coisas. Este é o retângulo na parte superior da tela que contém o botão Voltar e o título da tela. O retângulo inteiro é geralmente chamado de cabeçalho no React Navigation.

Screen component

Um componente de tela é um componente que usamos em nossa configuração de rota.

const Stack = createStackNavigator();

const StackNavigator = (
  <Stack.Navigator>
    <Stack.Screen
      name="Home"
      component={HomeScreen} // <----
    />
    <Stack.Screen
      name="Details"
      component={DetailsScreen} // <----
    />
  </Stack.Navigator>
);

The suffix Screen in the component name is entirely optional, but a frequently used convention; we could call it Michael and this would work just the same.

Vimos anteriormente que nossos componentes de tela são fornecidos com o prop navigation. It's important to note that this only happens if the screen is rendered as a route by React Navigation (for example, in response to navigation.navigate). Por exemplo, se renderizarmos DetailsScreen como um filho de HomeScreen, então DetailsScreen não será fornecido com o navigation prop e quando você pressionar o botão "Ir para detalhes... novamente "botão na tela inicial, o aplicativo irá lançar uma das excepções JavaScript" undefined não é um objeto ".

function HomeScreen() {
  return (
    <View style={{ flex: 1, alignItems: 'center', justifyContent: 'center' }}>
      <Text>Home Screen</Text>
      <Button
        title="Go to Details"
        onPress={() => navigation.navigate('Details')}
      />
      <DetailsScreen />
    </View>
  );
}

The "Navigation prop reference" section goes into more detail on this, describes workarounds, and provides more information on other properties available on navigation prop.

Navigation Prop

Este prop será transmitido para todas as telas e pode ser usado para o seguinte:

  • dispatch enviará uma ação até o roteador
  • navigate, goBack, etc estão disponíveis para despachar ações de maneira conveniente

Navegadores também podem aceitar um suporte de navegação, que eles devem obter do navegador pai, se houver um.

Para mais detalhes, veja o "Navigation prop document".

The "Route prop reference" section goes into more detail on this, describes workarounds, and provides more information on other properties available on route prop.

Route prop

This prop will be passed into all screens. Contains information about current route i.e. params, key and name.

Navigation State

O estado de um navegador geralmente é algo como isto:

{
  key: 'StackRouterRoot',
  index: 1,
  routes: [
    { key: 'A', name: 'Home' },
    { key: 'B', name: 'Profile' },
  ]
}

Para este estado de navegação, há duas rotas (que podem ser tabs, ou cards em um stack). O índice indica a rota ativa, que é "B".

Route

Each route is a piece of navigation state which contains a key to identify it, and a "name" to designate the type of route. Também pode conter parâmetros arbitrários:

{
  key: 'B',
  name: 'Profile',
  params: { id: '123' }
}
