Custom navigators

A navigator is any React component that has a router on it, to define the navigation behavior. Cada navegador recebe um prop navigation, que permite ao pai controlar o estado de navegação.

Estendendo navegadores

É possível ter um navegador existente e estender seu comportamento, usando a seguinte abordagem:

const MyStack = createStackNavigator({ ... });

class CustomNavigator extends React.Component {
  static router = MyStack.router;
  render() {
    const { navigation } = this.props;

    return <MyStack navigation={navigation} />;
  }
}

Agora é possível renderizar coisas adicionais, observar o suporte da navegação e substituir o comportamento do roteador:

const MyStack = createStackNavigator({ ... });

class CustomNavigator extends React.Component {
  static router = {
    ...MyStack.router,
    getStateForAction: (action, lastState) => {
      // check for custom actions and return a different navigation state.
      return MyStack.router.getStateForAction(action, lastState);
    },
  };
  componentDidUpdate(lastProps) {
    // Navigation state has changed from lastProps.navigation.state to this.props.navigation.state
  }
  render() {
    const { navigation } = this.props;

    return (
      <View>
        <MyStack navigation={navigation} />
        {...}
      </View>
    );
  }
}

Navegador Navegação Prop

O elemento de navegação transmitido para um navegador inclui apenas state, dispatch, e addListener. Esse é o estado atual do navegador e um canal de eventos para enviar solicitações de ação.

Todos os navegadores são componentes controlados: eles sempre exibem o que está chegando através de props.navigation.state, e a única maneira de alterar o estado é enviar ações para props.navigation.dispatch.

Os navegadores podem especificar o comportamento personalizado para os navegadores pai personalizando seu roteador. Por exemplo, um navegador é capaz de especificar quando as ações devem ser bloqueadas retornando null de router.getStateForAction. Ou um navegador pode especificar o processamento de URI personalizado sobrescrevendo router.getActionForPathAndParams para gerar uma ação de navegação relevante e manipular essa ação em router.getStateForAction.

Navigation State

O estado de navegação que é passado para o props.navigation.state do navegador tem a seguinte estrutura:

{
  index: 1, // identifies which route in the routes array is active
  routes: [
    {
      // Each route needs a name, which routers will use to associate each route
      // with a react component
      routeName: 'MyRouteName',

      // A unique id for this route, used to keep order in the routes array:
      key: 'myroute-123',

      // Routes can have any additional data. The included routers have `params`
      ...customRouteData,
    },
    ...moreRoutes,
  ]
}

Despachantes de Navegação

Um navegador pode despachar ações de navegação, como "Ir para um URI", "Voltar".

O dispatcher retornará true se a ação foi tratada com sucesso, caso contrário, false.

API para construir navegadores personalizados

Para ajudar os desenvolvedores a implementar navegadores personalizados, os seguintes utilitários são fornecidos com o React Navigation:

createNavigator

Nota: A API createNavigator foi alterada na versão 2. O documento antigo para v1 pode ser acessado aqui: https://v1.reactnavigation.org/docs/custom-navigators.html

Este utilitário combina um roteador e uma exibição de navegação juntos de uma maneira padrão:

import {createNavigator} from 'react-navigation';

const AppNavigator = createNavigator(NavigationView, router, navigationConfig);

O novo AppNavigator pode ser renderizado como:

<AppNavigator
  navigation={{ state, dispatch, addListener }}
  screenProps={...}
/>

Então o View será renderizada da seguinte maneira:

<NavigationView
  screenProps={screenProps}
  navigation={navigation}
  navigationConfig={navigationConfig}
  descriptors={descriptors}
/>

A prop navigation é o mesmo objeto de navegação que foi passado para o navegador.

Ambos navigationConfig e screenProps são simplesmente passados, conforme definido acima.

A maioria das informações sobre telas filhas vem através do descritor descriptors. Os descritores são um mapa de objeto de chaves de rota para descritores de cena. Existe um descritor para cada rota filha.

Descritores de cena

Um descritor de cena tem as seguintes propriedades:

  • getComponent, uma função que retorna o componente para a tela
  • options, as opções de navegação para a rota
  • navigation, o suporte à navegação filha, incluindo ações e o caminho state
  • state, o estado de navegação para a tela filha (um atalho para navigation.state)
  • key, a chave da rota (um atalho para navigation.state.key)

createNavigationContainer

If you want your navigator to be usable as a top-level component, (without a navigation prop being passed in), you can use createNavigationContainer. This utility will make your navigator act like a top-level navigator when the navigation prop is missing. It will manage the app state, and integrate with app-level nav features, like handling incoming and outgoing links, and Android back button behavior.
