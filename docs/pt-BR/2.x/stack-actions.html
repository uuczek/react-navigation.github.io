StackActions reference

StackActions é um objeto que contém métodos para gerar ações específicas para navegadores baseados em "Stack". Its methods expand upon the actions available in NavigationActions.

As seguintes ações são suportadas: * Reset - Substituir o estado atual por um novo estado * Replace - Substituir uma rota em uma determinada chave por outra rota * Push - Adicione uma rota no topo da "Stack" e navegue para frente * Pop - Navegar de volta para as rotas anteriores * PopToTop - Navegue até a rota superior da "Stack", dispensando todas as outras rotas

reset

A ação reset apaga todo o estado de navegação e o substitui pelo resultado de várias ações.

  • index - number - requerido - Índice da rota ativa dentro do array routes no state do navegador.
  • actions - array - requerido - Array de Ações do Navegador que substituirá o estado do navegador.
  • key - string or null - opcional - Se definido, o navegador com a chave especificada será resetado. Se null, o navegador raiz será resetado.
import { StackActions, NavigationActions } from 'react-navigation';

const resetAction = StackActions.reset({
  index: 0,
  actions: [NavigationActions.navigate({ routeName: 'Profile' })],
});
this.props.navigation.dispatch(resetAction);

Como usar o parametro index

O parâmetro index para especificar a rota ativa.

exemplo: dada uma navegador básico de "Stack" com duas rotas Profile e Settings. Para resetar o estado para um ponto em que a tela ativa era Settings, mas tê-lo empilhado em cima de uma tela Profile, você faria o seguinte:

import { StackActions, NavigationActions } from 'react-navigation';

const resetAction = StackActions.reset({
  index: 1,
  actions: [
    NavigationActions.navigate({ routeName: 'Profile' }),
    NavigationActions.navigate({ routeName: 'Settings' }),
  ],
});
this.props.navigation.dispatch(resetAction);

replace

A ação replace substitui a rota na chave dada por outra rota.

  • key - string - Key da rota para substituir. Se isso não estiver definido, a rota mais recente será substituída.
  • newKey - string - Key para usar na rota de substituição. Gerado automaticamente se não for fornecido.
  • routeName - string - routeName usada como rota substituta.
  • params - object - Parâmetros para passar para a rota substituta.
  • action - object - Sub-ação opicional.
  • immediate* - boolean - * Atualmente não tem efeito, este é um espaço reservado para quando o "Stack Navigator" suporta a substituição animada (atualmente não).

push

A ação push adiciona uma rota no topo da "Stack" e navega para frente. Isso difere de navigate em que navigate irá retornar ao início na "Stack" se uma rota do nome já estiver presente lá. O push sempre será adicionado por cima, então uma rota pode estar presente várias vezes.

  • routeName - string - routeName para empurrar para a "Stack".
  • params - object - Parâmetros de tela para mesclar a rota de destino (encontrado na tela pressionada através de this.props.navigation.state.params).
  • action - (avançado) A sub-ação para executar no "router" filho, se a tela for um navegador.
import { StackActions } from 'react-navigation';

const pushAction = StackActions.push({
  routeName: 'Profile',
  params: {
    myUserId: 9,
  },
});

this.props.navigation.dispatch(pushAction);

pop

A ação pop leva você de volta a uma tela anterior na "Stack". O parametro n permite que você especifique quantas telas aparecerão.

  • n - number - O número de telas para aparecer novamente.
import { StackActions } from 'react-navigation';

const popAction = StackActions.pop({
  n: 1,
});

this.props.navigation.dispatch(popAction);

popToTop

A ação popToTop leva você de volta à primeira tela da "Stack", descartando todos os outros. É funcionalmente idêntico ao StackActions.pop({n: currentIndex}).

import { StackActions } from 'react-navigation';

this.props.navigation.dispatch(StackActions.popToTop());
