在 Web 上使用React Navigation

从 v3 开始，React Navigation 内置支持网站开发，包括服务端渲染。

在一个 React 应用中设置导航, (例如一个使用 create-react-app 创建的项目):

import { createSwitchNavigator } from "@react-navigation/core";
import { createBrowserApp } from "@react-navigation/web";

const MyNavigator = createSwitchNavigator(routes);

const App = createBrowserApp(MyNavigator);

// now you can render "App" normally

Web Links

我们开箱即用，它会使用正确的href </ code>自动为您设置 <code><a>标签。

这对于正确支持服务器渲染是必要的，对于可访问性至关重要，并且当浏览器显示链接将要访问的URL时，可以提供良好的用户体验。

当应用程序运行时，将阻止默认的浏览器行为，并将调度导航操作。

To render a link to the "Profile" route: 渲染一个指向 Profile 路由的链接。

<Link toRoute="Profile" params={{ name: "jamie" }}>
  Jamie's Profile
</Link>

根据为Profile </ code>路由设置的<code> path </ code>，上述链接可能会呈现的 html 为 <code><a href="/people/jamie">Jamie's Profile</a>

您也可以为`` Link </ code>提供 action </ code> prop，以指定将用于处理此链接的确切导航操作。

服务端渲染

您可以使用 handleServerRequest 函数获取应用程序的顶级导航道具，以及此路由的当前标题。

expressApp.get("/*", (req, res) => {

const { path, query } = req;


const { navigation, title, options } = handleServerRequest(
AppNavigator.router,
path,
query
);


const markup = renderToString();


res.send(
`<!doctype html>


${title}



  
    
${markup}

  
`
  );
});
``
