createMaterialBottomTabNavigator

屏幕底部的材料设计主题标签栏，可让您在不同路由之间切换。 路由被懒加载 - 它们的屏幕组件直到第一次获取焦点时才被加载。

要使用此导航器，您需要安装react-navigation-material-bottom-tabs

npm install react-navigation-material-bottom-tabs react-native-paper

此API还要求您安装react-native-vector-icons！ 如果你正在使用 Expo，可以开箱即用。 否则，请按照这些安装说明来操作。

createMaterialBottomTabNavigator(RouteConfigs, MaterialBottomTabNavigatorConfig);

This library uses the BottomNavigation component from react-native-paper. It doesn't include the whole react-native-paper library in your bundle, so you don't need to worry about it.

RouteConfigs

路由配置对象是从路由名称到路由配置的映射。

MaterialBottomTabNavigatorConfig

  • shifting - 无论使用移位样式，活动选项卡都显得更宽，而非活动选项卡不会有标签。 默认情况下，当您有三个以上的选项卡时，这是正确的。
  • labeled - 是否在标签中显示标签。 当false时，只会显示图标。
  • activeColor - Custom color for icon and label in the active tab.
  • inactiveColor - Custom color for icon and label in the inactive tab.
  • barStyle </ code> - 底部导航栏的样式。 如果您在Android上有半透明导航栏，则可以在此处设置底部填充：<code> barStyle = {{paddingBottom: 48}}。
  • initialRouteName -第一次加载时初始选项卡路由的 routeName。
  • order -定义选项卡顺序的 routeNames 数组。
  • paths - 提供 routeName 到 path 配置的映射, 它重写 routeConfigs 中设置的路径。
  • backBehavior - 控制 "返回" 按钮是否会导致 Tab 页切换到初始 Tab 页? 如果是, 设置为 initialRoute, 否则 none。 默认为 initialRoute的行为。

例子:

export default createMaterialBottomTabNavigator({
  Album: { screen: Album },
  Library: { screen: Library },
  History: { screen: History },
  Cart: { screen: Cart },
}, {
  initialRouteName: 'Album',
  activeColor: '#f0edf6',
  inactiveColor: '#3e2465',
  barStyle: { backgroundColor: '#694fad' },
});

用于导航器内部页面的navigationOptions

标题

可用作headerTitle和tabBarLabel的后备的通用标题。

tabBarIcon

React Element or a function that given { focused: boolean, tintColor: string } returns a React.Node to display in tab bar.

tabBarColor

Color for the tab bar when the tab corresponding to the screen is active. Used for the ripple effect. This is only supported when shifting is true.

tabBarLabel

Title string of a tab displayed in the tab bar. When undefined, scene title is used. To hide, see labeled option in the previous section.

tabBarAccessibilityLabel

选项卡按钮的辅助功能标签。 当用户点击标签时，屏幕阅读器会读取这些信息。 如果您没有选项卡的标签，建议设置此项。

tabBarTestID

用于在测试中找到该选项卡按钮的 ID。

tabBarOnPress

处理点击事件的回调; 该参数是一个对象，其中包含：

  • navigation：页面的 navigation props
  • defaultHandler: tab press 的默认 handler

在开始转换到下一个场景之前添加自定义逻辑（点击的场景）。

Using with react-native-paper (optional)

You can use the theming support in react-native-paper to customize the material bottom navigation by wrapping your app in Provider from react-native-paper. A common use case for this can be to customize the background color for the screens when your app has a dark theme. Follow the instructions on react-native-paper's documentation to learn how to customize the theme.
