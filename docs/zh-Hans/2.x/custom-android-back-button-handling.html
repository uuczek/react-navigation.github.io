Custom Android back button behavior

默认情况下，当用户按下 Android 物理返回键时，react-navigation 会返回到新页面，如果没有可返回的页面，则退出应用。 这是一个合理的默认行为，但有些情况下你可能想要实现自定义处理。

如果你正在寻找易于使用的解决方案，则可以查看社区开发的软件包 react-navigation-backhandler。 以下内容将向你展示这个 package 表面下的实际工作原理。

例如，在一个页面中，用户正在列表中选择某一项，并且“选择模式”处于活跃状态。 在按下返回按钮时，你首先需要停用“选择模式”，然后在第二次按下返回键时，返回上一页。 以下代码片段演示了这种情况。 我们利用 react-native 提供的BackHandler这个 Api，并 订阅navigation 的生命周期方法 以添加我们自定义的hardwareBackPress事件监听器。

从onBackButtonPressAndroid方法返回true表示我们已经处理了该事件，并且 react-navigation 的事件监听器不会被调用，因此不会销毁当前页。 返回false会该方法继续执行 - react-navigation 的事件监听器将销毁当前页面。

import React from "react";
import { BackHandler } from "react-native";

class ScreenWithCustomBackBehavior extends React.Component {
  _didFocusSubscription;
  _willBlurSubscription;

  constructor(props) {
    super(props);
    this._didFocusSubscription = props.navigation.addListener('didFocus', payload =>
      BackHandler.addEventListener('hardwareBackPress', this.onBackButtonPressAndroid)
    );
  }

  componentDidMount() {
    this._willBlurSubscription = this.props.navigation.addListener('willBlur', payload =>
      BackHandler.removeEventListener('hardwareBackPress', this.onBackButtonPressAndroid)
    );
  }

  onBackButtonPressAndroid = () => {
    if (this.isSelectionModeEnabled()) {
      this.disableSelectionMode();
      return true;
    } else {
      return false;
    }
  };

  componentWillUnmount() {
    this._didFocusSubscription && this._didFocusSubscription.remove();
    this._willBlurSubscription && this._willBlurSubscription.remove();
  }

  render() {
    // ...
  }
}

所提出的方法对于显示在StackNavigator中的页面很适用。 在某些情况下，自定义返回键处理可能暂时不被支持（例如， 目前已知的一种情况就是：在打开的 drawer 中进行返回按钮的事件处理。 如若遇到类似情况，欢迎给我们提 PR）。

为什么不使用组件的生命周期方法？

起初，你可能倾向于使用componentDidMount来订阅返回事件，componentWillUnmount取消订阅。 This approach will not work - learn more about this in navigation lifecycle.
