Using the navigation key

key参数在不同的导航功能中重复出现。 我们来看看它的用例总结：

navigate 的调用方法

如果没有提供 key，StackRouter的行为如下：

  • 如果已经存在具有给定名称的路由，StackRouter将跳转到现有路由，同时设置新参数。
  • 如果不存在这样的路由，StackRouter会将它推入堆栈

但是，如果您要推送相同路由的多个实例，则每次调用navigate时都可以通过提供唯一的key参数来实现此操作，或者可以使用 StackRouter上的push操作。 有关更多背景信息，请参阅相关的 RFC 。

注意：在1.x系列版本中，没有key的navigate行为显着不同。 在这儿. 阅读更多内容

reset 的用法

重置时，key也是可选的，可以是字符串或null。 如果设置，具有给定 key 的导航器将重置。 如果null，则根导航器将重置。 您可以通过调用this.props.navigation.dangerouslyGetParent().state.key。 来获取路由的 navigator key。 函数被称为 dangerouslyGetParent 的原因是警告开发人员不要过多使用它。 获取父级的父级以及其他难以遵循的模式。

replace 的用法

在replace导航操作时，key是用于标识要替换的路由的必需参数。 如果使用助手函数this.props.navigation.replace，我们将自动替换当前路由的键。
