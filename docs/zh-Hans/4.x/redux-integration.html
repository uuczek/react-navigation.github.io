Redux integration

在带有React Navigation的应用程序中使用Redux非常容易。 与没有React Navigation基本上没有什么不同。 以下示例显示了如何从头到尾进行操作：https://snack.expo.io/@react-navigation/redux-example。 其中最重要的部分是：

let RootStack = createStackNavigator({
  Counter: CounterContainer,
  StaticCounter: StaticCounterContainer,
});

let Navigation = createAppContainer(RootStack);

// Render the app container component with the provider around it
export default class App extends React.Component {
  render() {
    return (
      <Provider store={store}>
        <Navigation />
      </Provider>
    );
  }
}

请注意，我们采用了从createAppContainer返回的组件，并将其包装在Provider中。 哒哒! 现在，随时可以在整个应用中使用connect。

那么navigationOptions呢？

很好，这不是最明显的答案。 假设你要从标题访问Redux存储状态，该怎么办？ 有两种选择。 对于这些示例，假设要将上述示例中的计数放入标题中。

使用connect连接的组件

创建一个组件，将其connect到 store，然后在title中使用该组件。

class Count extends React.Component {
  render() {
    return <Text>Count: {this.props.value}</Text>;
  }
}

let CountContainer = connect(state => ({ value: state.count }))(Count);

class Counter extends React.Component {
  static navigationOptions = {
    title: <CountContainer />,
  };

  /* .. the rest of the code */
}

看一个可运行的示例.

将你关心的 state 作为参数传递给页面

如果不希望更改该值，则可以将其从connect组件传递到另一个页面作为参数。

<Button
  title="Go to static count screen"
  onPress={() =>
    this.props.navigation.navigate('StaticCounter', {
      count: this.props.count,
    })
  }
/>

class StaticCounter extends React.Component {
  static navigationOptions = ({ navigation }) => ({
    title: navigation.getParam('count'),
  });

  render() {
    return (
      <View style={styles.container}>
        <Text style={styles.paragraph}>
          {this.props.navigation.getParam('count')}
        </Text>
      </View>
    );
  }
}

看一个可运行的示例.

在页面上 setParams

让我们修改上一个示例中的StaticCounter，如下所示：

class StaticCounter extends React.Component {
  static navigationOptions = ({ navigation }) => ({
    title: navigation.getParam('count'),
  });

  componentDidMount() {
    this.updateCount();
  }

  componentDidUpdate() {
    this.updateCount();
  }

  updateCount() {
    this.props.navigation.setParams({ count: this.props.count });
  }

  render() {
    return (
      <View style={styles.container}>
        <Text style={styles.paragraph}>
          {this.props.navigation.getParam('count')}
        </Text>
      </View>
    );
  }
}

现在，无论何时 store 更新，我们都会更新count参数，标题也会相应更新。

我也可以在Redux中存储导航状态吗？

从技术上讲，这是可行的，但我们不建议这样做-太容易使自己陷入困境，放慢/破坏你的应用程序了。我们鼓励您将其留给React Navigation来管理导航状态。 但是，如果确实要执行此操作，则可以使用 react-navigation-redux-helpers ，但这不是官方支持的工作流程。
