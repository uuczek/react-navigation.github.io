Transitioner

Transitioner是一个 React 组件，可帮助管理复杂动画组件的切换。 它管理动画的时间，并在进入和离开时跟踪各种页面，但它不知道页面会呈现什么样子，因为渲染完全是交给给开发人员的。

事实上，Transitioner用于实现CardStack，因此也是 stack navigator。

Transitioner做的最有用的一件事，就是引入了当前 navigation state。 当从该 navigation state 中删除路由时，Transitioner会协调从这些路由消失的过渡动画，即使它们必须从 navigation state 中消失，也要将它们保持在页面上。

例子

class MyNavView extends Component {
  ...
  render() {
    return (
      <Transitioner
        configureTransition={this._configureTransition}
        navigation={this.props.navigation}
        render={this._render}
        onTransitionStart={this.onTransitionStart}
        onTransitionEnd={this.onTransitionEnd}
      />
    );
}

For a small but complete working example, see the CustomTransitioner example from the NavigationPlayground.

Props

configureTransition function

在Transitioner.componentWillReceiveProps上调用时，此函数允许自定义动画参数，例如duration。 从该函数返回的值将作为其配置输入到定时函数中，默认情况下是Animated.timing()。

例子

_configureTransition(transitionProps, prevTransitionProps) {
  return {
    // duration in milliseconds, default: 250
    duration: 500,
    // An easing function from `Easing`, default: Easing.inOut(Easing.ease)
    easing: Easing.bounce,
  }
}

注意：duration和easing仅适用于定时函数为Animated.timing的情况。 我们也可以使用不同的定时函数及其相应的配置参数，如下所示：

_configureTransition(transitionProps, prevTransitionProps) {
  return {
    // A timing function, default: Animated.timing.
    timing: Animated.spring,
    // Some parameters relevant to Animated.spring
    friction: 1,
    tension: 0.5,
  }
}

流程定义

  configureTransition: (
    transitionProps: NavigationTransitionProps,
    prevTransitionProps: ?NavigationTransitionProps,
  ) => NavigationTransitionSpec,

参数

  • transitionProps: the current NavigationTransitionProps created from the current navigation state and props
  • prevTransitionProps: the previous NavigationTransitionProps created from the previous navigation state and props

返回值

  • An object of type NavigationTransitionSpec that will be fed into an Animated timing function as its config

navigation prop

表示 navigation state 的对象，带有routes和一个活动路由index。 还包括dispatch和其他请求操作的方法。

示例值

{
   // Index refers to the active child route in the routes array.
  index: 1,
  routes: [
    { key: 'DF2FGWGAS-12', routeName: 'ContactHome' },
    { key: 'DF2FGWGAS-13', routeName: 'ContactDetail', params: { personId: 123 } }
  ]
}

流程定义

export type NavigationState = {
  index: number,
  routes: Array<NavigationRoute>,
};

For more information about the NavigationRoute type, check out its flow definition.

render function

从Transitioner.render()调用。 该函数执行从Transitioner委派的实际渲染。 在这个函数中，我们可以使用包含在transitionProps和prevTransitionProps参数中的信息来渲染场景，创建动画和处理手势。

transitionProps和prevTransitionProps参数的一些重要属性对上述任务很有用：

  • scenes: Array<NavigationScene> - 所有可用场景的列表
  • position：NavigationAnimatedValue - 转换器导航状态的渐进索引
  • progress: NavigationAnimatedValue - the value that represents the progress of the transition when navigation state changes from one to another. Its numeric value will range from 0 to 1.

For the complete list of properties of NavigationTransitionProps, check out its flow definition.

例子

transitionProps.scenes是所有可用场景的列表。 由实现者决定如何将它们放置在页面上。 例如，我们可以将场景渲染为一堆 Card，如下所示：

_render(transitionProps, prevTransitionProps) {
  const scenes = transitionProps.scenes.map(scene => this._renderScene(transitionProps, scene));
  return (
    <View style={styles.stack}>
      {scenes}
    </View>
  );
}

然后，我们可以使用Animated.View来为设置转换动画。 为了创建必要的动画样式属性（如opacity），我们可以对position和transitionProps附带的progress值进行空间插值：

_renderScene(transitionProps, scene) {
  const { position } = transitionProps;
  const { index } = scene;
  const opacity = position.interpolate({
    inputRange: [index-1, index, index+1],
    outputRange: [0, 1, 0],
  });
  // The prop `router` is populated when we call `createNavigator`.
  const Scene = this.props.router.getComponent(scene.route.routeName);
  return (
    <Animated.View style={{ opacity }}>
      { Scene }
    </Animated.View>
  )
}

上面的代码在转换过程中创建淡入淡出动画。

有关如何创建自定义切换的综合教程，请参阅此博文。

流程定义

render: (transitionProps: NavigationTransitionProps, prevTransitionProps: ?NavigationTransitionProps) => React.Node,

参数

  • transitionProps: the current NavigationTransitionProps created from the current state and props
  • prevTransitionProps: the previous NavigationTransitionProps created from the previous state and props

返回值

  • ReactElement，将用于渲染 Transitioner 组件

onTransitionStart函数

在过渡动画即将开始时调用。

如果你从onTransitionStart返回一个 promise，则过渡动画将在 promise resolve 之后开始。

流程定义

onTransitionStart: (transitionProps: NavigationTransitionProps, prevTransitionProps: ?NavigationTransitionProps) => (Promise | void),

参数

  • transitionProps: the current NavigationTransitionProps created from the current state and props
  • prevTransitionProps: the previous NavigationTransitionProps created from the previous state and props

返回值

  • Promise用于延迟过渡动画的开始，或者不立即开始过渡动画。

onTransitionEnd 函数

过渡动画完成后调用。

如果你从onTransitionEnd返回一个 promise，则在 promise resolve后，将开始执行排队的过渡动画。

流程定义

onTransitionEnd: () => void

参数

  • none.

返回值

  • none.
