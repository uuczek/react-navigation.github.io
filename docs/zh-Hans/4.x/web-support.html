React Navigation on the Web

Note: starting in v3, React Navigation has built-in support for use in web sites, including server rendering. This has not yet been widely used in production and we consider this feature to be experimental.

With react-native-web

"React Native for Web" makes it possible to run React Native components and APIs on the web using React DOM.

This approach allows you to reuse most of React Navigation on the web because React Native for Web maps React Native primitives like View, Text, and others to their equivalents on the web.

The easiest way to get started with this approach is to use to use the Expo CLI web support beta. More information on how to set this up in other projects will follow in the future, help on documenting it is also welcome!

With standard web tools

This approach requires that you rebuild the navigation views for your app (at least until the community builds out an alternative), but allows you to leverage routers and simple navigators that don't require views, like the switch navigator.

在一个 React 应用中设置导航, (例如一个使用 create-react-app 创建的项目):

import { createSwitchNavigator } from "@react-navigation/core";
import { createBrowserApp } from "@react-navigation/web";

const MyNavigator = createSwitchNavigator(routes);

const App = createBrowserApp(MyNavigator);

// now you can render "App" normally

Web Links

我们开箱即用，它会使用正确的href </ code>自动为您设置 <code><a>标签。

这对于正确支持服务器渲染是必要的，对于可访问性至关重要，并且当浏览器显示链接将要访问的URL时，可以提供良好的用户体验。

当应用程序运行时，将阻止默认的浏览器行为，并将调度导航操作。

To render a link to the "Profile" route: 渲染一个指向 Profile 路由的链接。

<Link toRoute="Profile" params={{ name: "jamie" }}>
  Jamie's Profile
</Link>

根据为Profile </ code>路由设置的<code> path </ code>，上述链接可能会呈现的 html 为 <code><a href="/people/jamie">Jamie's Profile</a>

您也可以为`` Link </ code>提供 action </ code> prop，以指定将用于处理此链接的确切导航操作。

服务端渲染

您可以使用 handleServerRequest 函数获取应用程序的顶级导航道具，以及此路由的当前标题。

expressApp.get("/*", (req, res) => {

const { path, query } = req;


const { navigation, title, options } = handleServerRequest(
AppNavigator.router,
path,
query
);


const markup = renderToString();


res.send(
`<!doctype html>


${title}



  
    
${markup}

  
`
  );
});
``
