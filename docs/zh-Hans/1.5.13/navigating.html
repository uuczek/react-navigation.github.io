Moving between screens

If you run this code, you'll notice that each time you press the "Go to Details... again" button it will push a new screen on top. This is where our original comparison to document.location.href falls apart, because in a web browser these would not be treated as distinct routes and no new entries would be added to the browser history â€” navigate for StackNavigator behaves more like the web's window.history.pushState: each time you call navigate it pushes a new route to the navigation stack.
